Prerequisites
- installed openvpn binary
- tun in kernel or tun.ko
OpenVPN Settings is being developed on current releases of CyanogenMod (6 or greater), which includes the openvpn binary and tun.

Usage
- To edit the config file and config dependent properties, touch config entry long (this will open the context menu).
Features
- since 0.4.1
- DNS support.
- Send mail to author
- since 0.2
- UI similar to Wi-Fi settings.
- Restart tunnel when network connectivity changes (e.g. from wifi to 3g).
- Start tunnel on boot, according to last state.
- Turn on/off openvpn service, saving individual tunnel state.
- Passphrase support.
- Username/Password support (untested, please report any problems).
- Uses Management Interface.
- Edit config file in text editor,
- Notification support.
- No log written to file system, to save your flash memory.
- Should support multiple independent tunnels (reported to work fine).

Missing Features
- Display log output.
- Display more details about ongoing connection in notification area.
Logging
Use adb logcat on your host. OpenVPN Settings currently relies heavily on adb logcat to give you all information about what's going on. This comes in handy especially when setting up a new tunnel.

Setup
1. Connect your phone via USB to your computer and mount the sdcard.
2. Create a directory named openvpn at the root of your sdcard.
3. Copy your certificates and *.conf files into that directory (The config files need to end in .conf or .ovpn ).
4. Unmount the sdcard. 5. You might need to hit refresh from the menu if you started OpenVPN Settings already.
6. Start adb logcat on your computer to monitor activity.
7. Open activity OpenVPN Settings and enable your tunnel.
In general it would be best to get your config running from a shell, e.g. using adb shell.

DNS Support
DNS Support is somewhat tricky to get right. Nevertheless it should work. If you experience hanging connections for no obvious reason, please try 'Fix DNS' from the main options menu. This will make a public DNS server the default. The next time network connectivity changes defaults provided by your ISP will be set. This is a new feature, please report any problems but also success. Links
